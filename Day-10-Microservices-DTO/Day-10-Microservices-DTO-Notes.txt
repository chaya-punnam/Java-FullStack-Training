# Day 10 - Spring Boot Microservices, DTO, Synchronous & Asynchronous Communication

## 1. What is a DTO?

DTO stands for **Data Transfer Object**.

A DTO is a simple Java class used to transfer data between different layers or different microservices.

### Why do we use DTO?

Without DTO:

```java
return DoctorEntity;
```

The entire entity data will be sent to the client.

Sometimes we do not want to expose all fields.

Example:

DoctorEntity contains:

* doctorId
* doctorName
* doctorAge
* doctorSalary
* doctorSpeciality
* hospitalId

But the client only needs:

* doctorId
* doctorName
* doctorAge
* doctorSpeciality

In this case DTO is used.

### Advantages of DTO

1. Hides unnecessary data.
2. Improves security.
3. Reduces response size.
4. Used for communication between microservices.
5. Makes APIs cleaner.

## 2. DTO in Doctor App

### DoctorDTO

```java
public class DoctorDTO {

    private Integer doctorId;
    private String doctorName;
    private int doctorAge;
    private String doctorSpeciality;

    private Integer hospitalId;
    private String hospitalName;
    private String hospitalLocation;
}
```

This DTO combines:

* Doctor Information
* Hospital Information

and sends them in a single response.

### Output

```json
{
  "doctorId": 1,
  "doctorName": "Ramu",
  "doctorAge": 31,
  "doctorSpeciality": "Neurology",
  "hospitalId": 1,
  "hospitalName": "Apollo Hospital",
  "hospitalLocation": "Hyderabad"
}
```

## 3. What is a Microservice?

A Microservice is a small independent application responsible for a specific business functionality.

Example:

### Doctor Service

Port: 8082

Responsibilities:

* Create Doctor
* Read Doctor
* Delete Doctor

### Hospital Service

Port: 8081

Responsibilities:

* Create Hospital
* Read Hospital
* Delete Hospital

Both applications run independently.

---

## 4. Communication Between Microservices

Doctor Service needs Hospital Details.

Doctor database stores only:

```java
hospitalId
```

To get complete hospital information:

Doctor Service calls Hospital Service.

### Using RestTemplate

```java
RestTemplate template = new RestTemplate();

HospitalDTO hospitalDetail =
template.getForObject(
"http://localhost:8081/hospital/" + hospitalId,
HospitalDTO.class);
```

Hospital Service returns hospital details.

Doctor Service combines both Doctor and Hospital data using DTO.

## 5. Synchronous Communication

Synchronous means:

The sender waits for the receiver's response.

### Flow

Doctor Service
↓
Calls Hospital Service
↓
Waits
↓
Gets Response
↓
Continues Execution

### Example

```java
RestTemplate template = new RestTemplate();

HospitalDTO hospital =
template.getForObject(
"http://localhost:8081/hospital/1",
HospitalDTO.class);
```

Doctor Service waits until Hospital Service responds.

### Examples of Synchronous Communication

1. REST API
2. Web Services (SOAP)
3. OpenFeign Client

### Advantages

* Easy to understand
* Easy to implement
* Immediate response

### Disadvantages

* Slow if another service is down
* Tight coupling
* One service depends on another

## 6. REST API

REST stands for:

Representational State Transfer

Used for communication between applications over HTTP.

### Common HTTP Methods

GET

Retrieve Data

POST

Insert Data

PUT

Update Data

DELETE

Delete Data

Example:

```java
GET /doctor/1
POST /doctor/create
DELETE /doctor/1
```

---

## 7. Asynchronous Communication

Asynchronous means:

The sender does not wait for the receiver.

The message is sent and processing continues.

### Flow

Doctor Service
↓
Sends Message
↓
Continues Execution
↓
Hospital Service Processes Later

No waiting.

### Advantages

* Faster
* Better performance
* Loose coupling
* Highly scalable

### Disadvantages

* More complex
* Difficult debugging

---

## 8. Apache Kafka

Kafka is a distributed event streaming platform.

Used for asynchronous communication.

### Components

Producer

Sends Messages

Consumer

Receives Messages

Topic

Stores Messages

### Flow

Producer
↓
Topic
↓
Consumer

Example:

Doctor Service sends:

"New Doctor Created"

Kafka stores it in a Topic.

Hospital Service consumes the message later.

### Benefits

* High Performance
* Fault Tolerance
* Real-Time Processing
* Scalable

---

## 9. RabbitMQ

RabbitMQ is a message broker.

Used for asynchronous communication.

### Flow

Producer
↓
Queue
↓
Consumer

Messages are stored in queues.

Consumers process them later.

### RabbitMQ vs Kafka

RabbitMQ

* Queue based
* Good for task processing

Kafka

* Topic based
* Good for real-time streaming

## Interview Questions

### What is DTO?

DTO stands for Data Transfer Object. It is used to transfer data between layers or microservices without exposing the entire entity.

### Why use DTO instead of Entity?

* Security
* Better API Design
* Smaller Response
* Hide Unnecessary Fields

### What is RestTemplate?

RestTemplate is a Spring class used to call external REST APIs.

### Difference Between Synchronous and Asynchronous Communication?

Synchronous:

* Waits for response
* Example: REST API

Asynchronous:

* Does not wait for response
* Example: Kafka, RabbitMQ

### Difference Between Kafka and RabbitMQ?

Kafka:

* Topic Based
* Event Streaming

RabbitMQ:

* Queue Based
* Message Processing
